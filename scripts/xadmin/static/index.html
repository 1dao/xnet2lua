关于 xadmin

            

            
xadmin 是基于 xnet2lua 的轻量管理控制台，通过 NATS 在多个 SERVER_NAME 进程间路由脚本执行。

            
    
                
  • 启动：bin/xnet scripts/xadmin/xadmin_main.lua SERVER_NAME=xadmin1
    • 
                
  • 账户登录：首次打开自动进入初始化页，配置 MySQL 连接与默认管理员；之后所有接口需登录会话。
    • 
                
  • 数据库配置保存在 xadmin_db.json（可用 XADMIN_DATA_FILE 覆盖路径），账户与会话存于 MySQL。
    • 
                
  • 也可在 xnet.cfg 用 XADMIN_DB_HOST/PORT/USER/PASSWORD/DATABASE 预置连接；有值即覆盖 JSON（按字段），五项齐全时初始化页只需设管理员账户。
    • 
                
  • 兼容旧版 XADMIN_TOKEN：设置后也可用 X-Xadmin-Token 头免登录调用接口。
    • 
                
  • mTLS：设置 XADMIN_HTTPS=1 + XADMIN_TLS_CA_FILE=<ca.pem> 后，访问控制台须出示由该 CA 签发的客户端证书；可在 oauth_links 表把 subject DN 映射到本地账户。
    • 
                
  • OAuth2：XADMIN_OAUTH_PROVIDERS=google,github + XADMIN_OAUTH_GOOGLE_{CLIENT_ID,CLIENT_SECRET,AUTH_URL,TOKEN_URL,USERINFO_URL} 启用 Auth Code + PKCE 登录。
    • 
                
  • JWT (HS256)：设 XADMIN_JWT_HS256_SECRET=<共享密钥> 启用（标准 RFC 7515 base64url 签名，可与上游网关共用同一密钥）；不设则禁用。可选 XADMIN_JWT_ISSUER / XADMIN_JWT_AUDIENCE 锁定来源。
    • 
                
  • 授权（角色）：口令会话、XADMIN_SUPER_TOKEN、XADMIN_TOKEN 为 admin；OAuth/JWT/mTLS 等联邦身份默认 viewer（只读），仅当其本地名出现在 XADMIN_ADMINS=user1,user2 或等于初始化管理员时才是 admin。/api/exec、/api/reload 仅限 admin。
    • 
                
  • 本地测试开关：XADMIN_DEV_NO_AUTH=1 一键关闭全部认证与授权——任何请求都按 admin 放行，跳过登录/JWT/OAuth/mTLS/角色门，无需配置数据库即可进控制台执行脚本。仅限本地，切勿用于生产。
